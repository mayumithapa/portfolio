# Contributing to My Portfolio

Thank you for interest in contributing to my portfolio repository! This is primarily a personal portfolio project, but contributions are welcome.

## 📋 Contribution Types

### 1. **Content Improvements**
- Fix typos or grammatical errors
- Improve descriptions or clarity
- Add better project explanations
- Update outdated information

### 2. **Design/UX Enhancements**
- Suggest visual improvements
- Propose better layouts
- Improve mobile responsiveness
- Enhance accessibility

### 3. **Documentation**
- Add development guides
- Improve setup instructions
- Clarify technical concepts
- Add usage examples

## 🚀 Getting Started

### Setup
```bash
git clone https://github.com/mayumithapa/portfolio.git
cd portfolio
```

### Making Changes

1. **Fork the repository** (if you don't have direct access)
2. **Create a branch**: `git checkout -b improvement/your-improvement`
3. **Make your changes**
4. **Commit with clear message**: `git commit -m "type: description"`
5. **Push your branch**: `git push origin improvement/your-improvement`
6. **Create a Pull Request** with description of changes

## 📝 Commit Message Format

```
<type>(<scope>): <subject>

<body>
```

**Types:**
- `docs`: Documentation updates
- `fix`: Bug fixes
- `style`: Formatting or styling
- `refactor`: Code improvements
- `feat`: New features

**Example:**
```
docs(readme): improve project descriptions

- Added more context to FlowForge-AI project
- Clarified architecture diagram explanations
- Updated performance metrics
```

## 📐 Code Style

- Keep it clean and readable
- Use consistent formatting
- Add comments for complex sections
- Follow existing patterns in the project

## 🔄 Review Process

1. **Automated checks** will run on your PR
2. I will **review your changes**
3. **Address any feedback** you receive
4. Once approved, your PR will be **merged**
